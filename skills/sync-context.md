# 文档回刷 SOP

**触发时机：** 每次功能实现或 Bug 修复完成后**必须**执行。

---

## 步骤

### 1. 读取功能状态注册表
读取 `docs/features.md`，了解当前状态。

### 2. 更新功能状态
根据本次完成的工作：
- 将已完成项从 📋（待做）或 🐛（缺陷）改为 ✅（完成）
- 修复的 Bug：将 🐛 移除或改为 ✅
- 部分修复：保留 🐛 并在备注中说明进展

**禁止：** 不得删除 TODO 项，只能改变其状态。

### 3. 新增条目（如有）
若本次新增了功能或 API 端点，在对应章节追加：
```
| 新功能名 | ✅ | 关键文件路径 | 简要说明 |
```

### 4. 更新 Sprint 待办
- 将已完成项从 Sprint 清单中标记为 ✅ 或移除
- 若发现新的 Bug 或优先级变化，在对应 Sprint 添加条目

### 5. 同步 AGENTS.md 优先级（如有变化）
若 Sprint 1 的最高优先级发生变化，更新 `AGENTS.md` 中「当前开发优先级」章节。

### 6. 输出变更摘要
完成后输出：
```
已更新 docs/features.md：
- [功能/Bug名] 状态：📋/🐛 → ✅
- 新增条目：xxx（如有）
```

---

## 快速参考：状态符号

| 符号 | 含义 |
|------|------|
| ✅ | 已完成，功能正常 |
| 📋 | 待实现 |
| 🐛 | 已实现但存在 Bug |
